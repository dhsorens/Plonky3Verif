# Keccak — Lean extraction (compiles)

Hax-extracted Lean for `p3-keccak`, set up to build with Lake. Mirrors
[`koala-bear`](../../../../koala-bear/proofs/lean/extraction/)'s structure
(extracted file + hand-written dependency stubs + lakefile), minus CompPoly.

## Build

```bash
cd keccak/proofs/lean/extraction
lake build            # compiles p3_keccak.lean against the stubs
# or, to regenerate p3_keccak.lean from Rust first, then build:
./build-lean.sh
```

`lake build` currently **succeeds** (with `sorry` warnings — see TCB below).

## Layout

| Path | Role |
|------|------|
| `p3_keccak.lean` | The **only** hax-generated file. Regenerated by `build-lean.sh`, then lightly post-processed (see below). |
| `p3_keccak/tiny_keccak.lean` | Stub for the external `tiny_keccak` crate (Keccak sponge + `keccakf`). Trusted placeholders. |
| `p3_keccak/p3_symmetric.lean` | Trait-class stubs (`Permutation`, `CryptographicPermutation`, `CryptographicHasher`) lifted from p3_symmetric's own extraction. |
| `p3_keccak/core_shims.lean` | Stubs for `core_models.*` items the pinned Hax proof-lib lacks (`Iterator.fold`, `u64::from/to_le_bytes`, range-slice indexing). |
| `p3_keccak/dependencies.lean` | Imports the three stub modules; `p3_keccak.lean` imports this. |
| `lakefile.toml`, `lean-toolchain`, `lake-manifest.json` | Lake package (lib `p3_keccak`; requires **Hax**, not CompPoly). |

## How `p3_keccak.lean` is produced (`build-lean.sh`)

1. **Scalar extraction.** `cargo hax into lean` is run with
   `-C target-feature=-sha3`, so keccak takes its portable scalar path
   (`tiny_keccak::keccakf`) instead of the aarch64 **NEON** module. The NEON path
   is unbuildable here: it needs `core_arch` SIMD intrinsics the proof-lib
   doesn't define, and emits `RustM.of_isOk (transmute …) (by rfl)` which can't
   typecheck (`transmute` is `opaque`). This drops the file from ~940 → ~240
   lines and removes those blockers.
2. **Two post-extraction rewrites** of the generated file (it's hax output; like
   koala-bear we post-process the *output*, never `keccak/src`):
   - inject `import p3_keccak.dependencies`;
   - delete the `(associatedTypes := { … with Item := … })` overrides — the hax
     engine (ad110bfa) emits an `IntoIterator<Item=…>` encoding whose `Item`
     field doesn't exist in the pinned Hax proof-lib (an engine ↔ proof-lib
     version skew). Dropping it lets the bound infer.

## Trusted base / `sorry`

This is a **type-checks**, not a **verified**, artifact. Outstanding `sorry`s:

- `p3_keccak.lean`: `KeccakF::permute` on `[[u64;1];25]` (fallback packed, uses an
  unsafe `&mut` transmute) and `Keccak256Hash::hash_iter` (hax can't translate
  its mutable-capture closure `apply_to_chunks(input, |buf| hasher.update(buf))`).
- `p3_keccak/*`: the stub bodies (`tiny_keccak` sponge, `from/to_le_bytes`,
  range-slice indexing) are placeholders. The Keccak permutation itself is **not
  modelled** yet.

So nothing here is proven; the value is that the extracted structure compiles and
gives a scaffold to fill in. CompPoly is intentionally not wired in yet.

## Dependency reuse

`.lake/` is gitignored. Locally it symlinks `.lake/packages` to koala-bear's
already-built packages to skip recompiling Hax. On a fresh checkout, delete the
symlink (if any) and run `lake build` normally — Lake will fetch Hax + Qq per
`lake-manifest.json`.
