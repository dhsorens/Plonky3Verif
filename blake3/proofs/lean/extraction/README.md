# Blake3 ‚Äî Lean extraction (compiles)

Hax-extracted Lean for `p3-blake3`, set up to build with Lake. Mirrors
[`keccak`](../../../../keccak/proofs/lean/extraction/)'s setup (extracted file +
hand-written dependency stubs + lakefile), which in turn mirrors `koala-bear`.

## Build

```bash
cd blake3/proofs/lean/extraction
lake build            # compiles p3_blake3.lean against the stubs
# or, to regenerate p3_blake3.lean from Rust first, then build:
./build-lean.sh
```

**Syncing with upstream:** re-extraction after an upstream merge is governed by
the repo-root [`SYNC.md`](../../../../SYNC.md); this crate's re-extract/reconcile
steps are in [`SYNC.md`](SYNC.md).

`lake build` currently **succeeds** (with `sorry` warnings ‚Äî see TCB below).

## Layout

| Path | Role |
|------|------|
| `p3_blake3.lean` | The **only** hax-generated file. Regenerated by `build-lean.sh`, then lightly post-processed (see below). |
| `p3_blake3/blake3.lean` | Stub for the external `blake3` crate (`Hasher`, `new`, `finalize`, `Hash`, `Into<[u8;32]>`). Trusted placeholders. |
| `p3_blake3/p3_util.lean` | Stub for `p3_util::apply_to_chunks` (no-op). |
| `p3_blake3/p3_symmetric.lean` | Trait-class stubs (`CryptographicHasher`, `Permutation`, ‚Ä¶); copied verbatim from keccak. |
| `p3_blake3/core_shims.lean` | `core_models.*` items the pinned Hax proof-lib lacks (here: just `Iterator.fold`). |
| `p3_blake3/dependencies.lean` | Imports the stub modules; `p3_blake3.lean` imports this. |
| `lakefile.toml`, `lean-toolchain`, `lake-manifest.json` | Lake package (lib `p3_blake3`; requires **Hax**, not CompPoly). |

## How `p3_blake3.lean` is produced (`build-lean.sh`)

1. **`cargo hax into lean`** (isolated `CARGO_TARGET_DIR` to avoid stale cached
   Lean). **No `-C target-feature` flag** ‚Äî unlike keccak, the `blake3` crate is
   external (crates.io) so its SIMD internals are never extracted; the wrapper
   only calls `blake3::Hasher::{new,update,finalize}`.
2. **Two post-extraction rewrites** of the generated file (it's hax output; like
   koala-bear we post-process the *output*, never `blake3/src`):
   - inject `import p3_blake3.dependencies`;
   - delete the `(associatedTypes := { ‚Ä¶ with Item := ‚Ä¶ })` overrides ‚Äî the hax
     engine emits an `IntoIterator<Item=‚Ä¶>` encoding whose `Item` field doesn't
     exist in the pinned Hax proof-lib (an engine ‚Üî proof-lib version skew).

## Trusted base / `sorry`

This is a **type-checks**, not a **verified**, artifact (see [`TCB.md`](TCB.md)).
The two `sorry`s in `p3_blake3.lean` are the `hash_iter` / `hash_iter_slices`
closure bodies, which hax can't translate (mutable capture / `&mut Self`-returning
`update`). The `blake3` crate itself and `apply_to_chunks` are unmodelled stubs,
so no BLAKE3 computation is represented yet. CompPoly is not wired in.

## Dependency reuse

`.lake/` is gitignored. Locally it symlinks `.lake/packages` to koala-bear's
already-built packages to skip recompiling Hax. On a fresh checkout, delete the
symlink (if any) and run `lake build` ‚Äî Lake will fetch Hax + Qq per
`lake-manifest.json`.
